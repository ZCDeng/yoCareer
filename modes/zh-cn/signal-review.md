# 模式：中文招聘信号审核

当用户要求审核 `data/signal-review.md` 中的招聘信号，或粘贴微博、小红书、公众号、群转发、内推、招聘帖内容时，按此流程处理。

## 目标

判断该信号应当：

- 晋升到 pipeline：`npm run signals -- promote --index N`
- 继续人工核验：保留在 `data/signal-review.md`
- 丢弃：`npm run signals -- discard --index N`

不得自动投递、自动私信、自动登录平台或批量联系招聘方。

## 审核维度

| 维度 | 判断问题 |
| --- | --- |
| HC 真实性 | 是否有公司、团队、岗位、职责、城市、时间、联系方式或官方链接 |
| 来源可靠性 | 官方账号、员工、招聘负责人、社区转发、匿名群消息分别标注 |
| 证据完整度 | 是否足以判断岗位真实存在；缺哪些信息 |
| 合同风险 | 外包、驻场、派遣、供应商、急招、高薪诚聘等风险 |
| 匹配度 | 是否符合 `cv.md`、`config/profile.yml`、`modes/_profile.md` |
| 行动成本 | 手动投递、私信、找内推、继续观察、丢弃 |

给出一个明确动作：

- `promote`

- `discard`

### 3. 建议命令

只在用户确认人工核验后建议命令。例如：

```bash
npm run signals -- draft --index 1
npm run signals -- promote --index 1
npm run signals -- discard --index 1
```

### 4. 手动沟通草稿

如果适合联系招聘方，生成一条短消息：

- 不超过 120 中文字。
- 不像群发。
- 说明一个具体匹配点。
- 不暴露敏感隐私。
- 不承诺虚假经历。

### 5. 核验问题

列出 3-5 个用户应该手动确认的问题，例如：

- 这是正式员工 HC 还是外包/驻场？
- 岗位在哪个城市/团队？
- 是否仍在招聘？
- JD 或官方投递链接是什么？
- 汇报线和业务方向是什么？
