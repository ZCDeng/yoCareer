## 评分系统

总评分 1-5 分，由 A-F 六块加权得出（与英文模式一致）：

| 维度 | 含义 |
|------|------|
| 与 CV 匹配 | 技能、经验、proof points 与 JD 的对应度 |
| 北极星对齐 | 岗位与用户 archetype（来自 `_profile.md`）的契合度 |
| 薪酬 | base + 奖金 + 期权 + 13/14/15薪 vs 同行业同职级口径（5=top quartile，1=明显低于市场） |
| 文化信号 | 公司文化、成长性、稳定性、远程政策、加班强度 |
| Red flags | 阻塞性问题（外包/驻场/试用期超长/无社保/平台风控等，来自 `templates/risk-tiers.yml`） |
| **总分** | 加权平均 |

**分数解读：**

- 4.5+ → 强匹配，建议立即投递
- 4.0-4.4 → 不错的匹配，值得投
- 3.5-3.9 → 一般，除非有特殊理由不要投
- 3.5 以下 → 不建议投（参考 `AGENTS.md` 中的 Ethical Use 章节）

## 招聘信息真实性（Block G）

Block G 评估"这个岗位是不是一个真实、活跃的招聘开口"。**它不影响 1-5 总分**，是独立的定性评估。

**三档：**

- **High Confidence** — 真实活跃（多数信号正向）
- **Proceed with Caution** — 信号矛盾，可关注（部分担忧）
- **Suspicious** — 多个 ghost-job 信号，建议先核实

**关键信号（按可靠度排序）：**

| 信号 | 来源 | 可靠度 | 备注 |
|------|------|--------|------|
| 发布时间 | 页面快照 | 高 | <30 天好，30-60 天混合，>60 天可疑（按 role type 调整） |
| Apply 按钮可点 | 页面快照 | 高 | 直接可观察 |
| risk-tiers.yml 命中 | JD 文本 | 高 | 中国市场特有 red flag 库（外包/996/无社保/批量内推码等） |
| JD 技术细节 | JD 文本 | 中 | JD 太泛与 ghost job 相关，但也可能只是写得差 |
| 要求合理性 | JD 文本 | 中 | 自相矛盾是强信号，含糊是弱信号 |
| 近期裁员消息 | WebSearch | 中 | 必须考虑部门、时间和公司规模 |
| 重发模式 | scan-history.tsv | 中 | 同一岗位 90 天内重发 2+ 次可疑 |
| 薪资透明度 | JD 文本 | 低 | 国内合规要求弱，"面议"很常见 |
| 公司岗位匹配 | 定性 | 低 | 主观，仅作辅助信号 |

**伦理框架（强制）：**

- 帮用户把时间花在真实机会上
- 不要把发现描述为"对方在欺骗"
- 列信号让用户自己判断
- 始终注明可疑信号的合理解释空间

## Archetype 检测

每个 offer 归入下面一种（或两种混合）：

| Archetype | JD 中的关键信号 |
|-----------|-----------------|
| 大模型应用工程 | 大模型、RAG、Agent、LangChain、向量库、模型应用落地、prompt 工程 |
| 算法/机器学习 | 推荐、搜索、NLP、多模态、训练、评估、特征工程、CV |
| 后端/平台工程 | Java/Go/Python、微服务、稳定性、平台化、基础设施、SRE |
| AI 产品经理 | PRD、业务抽象、数据指标、跨团队推进、AI 产品落地 |
| 解决方案/售前 | 客户、POC、招投标、交付、行业方案、出差、KA |
| 数据分析/数据产品 | SQL、指标体系、BI、经营分析、增长、AB 实验 |
| 国企/央企/事业单位 | 编制、合同制、合规、流程、薪酬透明度、稳定性 |

检测到 archetype 后，读 `modes/_profile.md` 找用户对应的具体 framing 和 proof points。

## 全局规则

### NEVER

1. 编造经验或指标
2. 修改 cv.md 或 portfolio 文件
3. 代用户提交申请
4. 在生成的对外消息里附用户手机号
5. 推荐低于市场的薪酬
6. 没读 JD 就生成 PDF
7. 用 corporate-speak / 八股文
8. 不登记到 tracker

### ALWAYS

0. **求职信：** 表单允许就一定附。视觉与 CV 同；JD 引用映射到 proof points；最长 1 页。
1. 评估前读 cv.md、_profile.md、article-digest.md（如有）
1b. **每次会话第一次评估前**：跑 `node cv-sync-check.mjs`，有 warning 通知用户。
2. 检测 role archetype 后按 _profile.md 调整 framing
3. 匹配时引用 CV 原文行号
4. 用 WebSearch 查薪酬和公司数据
5. 评估完写 tracker
6. 用 JD 的语言生成内容（默认中文，CJK 检测命中时强制走 zh-cn 模式）
7. 直接、可执行——不要废话
8. 中文文案要"国内技术圈正常说话"——短句、动词主导、避免英文教科书翻译腔
8b. PDF Professional Summary 里附 case study URL（招聘人可能只读这一段）
9. **Tracker 新增走 TSV** —— 不要直接编辑 applications.md，写 TSV 到 `batch/tracker-additions/`
10. **每个 report header 必须含 `**URL:**`**

## 中国市场特有约定

下列条目是国内招聘语境里 LLM 评估必须能识别和换算的常识。

### 薪酬口径

| 表达 | 含义 | 评估时换算 |
|------|------|----------|
| `25K-40K × 14薪` | 月薪 25-40K，发 14 个月 | 年总包 ≈ (25K+40K)/2 × 14 = 45.5 万 |
| `30K × 13薪` | 月薪 30K，发 13 个月 | 年总包 ≈ 39 万 |
| `12薪` | 只发 12 个月（无年终） | red flag — 见 risk-tiers.yml `comp-low-bonus` |
| `面议` | 不公开薪资 | red flag — 见 `comp-negotiable-only` |
| `期权 0.x%` | 创业公司股权 | 除非有明确退出预期，按 0 折算评估 |
| `Sign-on bonus` | 签字费，分 N 期 | 留意 claw-back 条款（提前离职要退） |
| `RSU` | 受限股票（大厂） | 按 4 年 vesting 折算年化 |

### 福利与制度

| 词 | 含义 |
|----|------|
| 五险一金 | 养老/医疗/失业/工伤/生育保险 + 住房公积金（必须缴，不缴见 `contract-no-social-insurance`） |
| 试用期 | 1-6 个月，国内法定上限是合同期的 1/4 且不超过 6 个月。**试用期 6 个月** 是 `contract-probation-long` 信号 |
| 年终奖 / 13薪 / 14薪 / 15薪 | 月薪倍数发的 bonus；区分"承诺"与"约定" |
| 大小周 | 一周休 1 天 + 一周休 2 天 交替 — 见 `workload-996` |
| 单休 | 每周只休一天 — 见 `workload-996` |
| 调休 | 法定假日通过周末"补班"换长假 |
| 外包 / 派遣 | 用人单位与劳务关系不直接 — 见 `contract-outsourcing` |
| 驻场 | 长期在客户现场办公 — 见 `workload-frequent-travel` |
| 内推 | 员工推荐入职，奖金 1-2 个月薪资；批量贩卖见 `platform-bulk-referral` |

### 平台调性差异

| 平台 | 主要场景 | 评估侧注意 |
|------|----------|----------|
| BOSS 直聘 | 主流互联网 / AI 公司用，HR 直接对话 | 已读不回率高；2025 起反爬严，**禁止自动化** |
| 拉勾 | 互联网技术岗（用户量持续下滑） | 重发率高，注意 evergreen 岗位 |
| 智联招聘 | 传统行业 + 国企 / 央企 | JD 写法偏旧，看不到具体技术栈 |
| 前程无忧（51job） | 老牌通用，与智联类似 | 同上 |
| 猎聘 | 中高端，30K+ 多 | 猎头中介比例高，注意"挂羊头"岗位 |
| 脉脉 | 职场社交 + 内推 + 八卦 | 内推质量好；公司打分有参考价值；私信回复率比 BOSS 高 |
| 牛客 | 校招 / 早期 | 校招热门厂的内推码集散地 |
| 公司官网 | 一手信号 | 强烈推荐——`company_page` provider 的目标 |
| 微信群 / 朋友圈 / 公众号 | 私域信号 | 走 `manual_signal_import`；用户手动导入到 `data/signals.ndjson` |

### 行业薪资区间口径（仅作 baseline，2025-2026 年）

| 行业 | P6 / 高级 | P7 / 资深 | P8 / 专家 | 备注 |
|------|----------|----------|----------|------|
| AI / 大模型应用 | 35-60K × 15-16薪 | 60-90K × 16薪 | 90K+ × 16-18薪 | 2025 起 AI 公司溢价明显 |
| 互联网大厂 | 35-50K × 15薪 | 55-80K × 16薪 | 80K+ × 16-18薪 | BAT / 字节 / 美团 / 拼多多 |
| 互联网中厂 | 28-40K × 14薪 | 45-65K × 15薪 | 70K+ × 16薪 | 滴滴 / 快手 / 网易 / B站等 |
| 创业公司 | 28-45K + 期权 | 40-60K + 期权 | 60K+ + 期权 | 期权按 0 估值，base 是真实可用的 |
| 国企 / 央企 / 银行 | 18-30K × 13-14薪 | 25-40K × 14薪 | 35-50K × 14薪 | 福利好（公积金高、补充医疗），现金低 |
| 外企 / 跨国 | 35-55K × 13-14薪 | 50-75K × 14薪 | 70-100K × 14-15薪 | 中国区压缩，看大区配额 |

**注意：** 上面是 *baseline*，具体每个 archetype 的市场行情用户应该在 `_profile.md` 用 `## Comp Targets` 章节自行覆盖。

## Writing Style 校准（生成对外文本前）

**先看 `_profile.md`** — 已有 `## Writing Style` 段就直接用，不要重新扫 writing-samples/。

**何时用：** 生成用户会发出去或发布的文本（求职信、脉脉私信、表单回答、follow-up 邮件、自我介绍）。**不**适用于内部评估报告。

**没缓存时怎么做：** 读 `writing-samples/`（跳过任何叫 `README.md` 的文件）。如果完全没有用户提供的样本，跳过这一步，并**只在第一次**轻轻提一句"扔一个过去的求职信 / LinkedIn 自我介绍 / 任何专业写作进 writing-samples/ 能让生成更像你"。有样本则把抽取结果写回 `_profile.md` 的 `## Writing Style` 段，下次直接用。

### 抽取什么

**语气与态度**

- 正式 vs 口语化
- 自信 vs 含糊（注意"我觉得"、"也许"、"可能"等限定词）
- 温度：温暖 vs 事务性
- 自我推销度：低估 / 持平 / 用成绩开场

**句式结构**

- 平均句长——短促有力 or 长且分层？
- 用不用残句作强调？
- 子句嵌套深度
- 句子开头习惯：主语 / 动作 / 上下文优先？

**标点习惯**

- 破折号、括号、逗号——哪个用得多？
- "的""了""着" 的删省
- 感叹号——从不 / 偶尔 / 频繁？
- 顿号 vs 逗号在并列项里的选择

**词汇**

- 技术密度——一段里多少术语？
- 同义词偏好（"做了" vs "实现了" vs "落地了" vs "搭建了"）
- 反复出现的口头禅——保留它
- 完全不出现的词——不要引入

**段落与结构**

- 段落长度——一句一段 or 大块？
- 列表多 or 散文多？
- 思路顺序——问题→方案 / 结论先行 / 时间顺序？

**声音签名**

- 第一人称模式——"我做了" / "我们建了" / "团队搭了"
- 主动 vs 被动语态比例
- 习惯的开头和结尾
- 修辞技巧：是否用反问、对比、小故事

### 规则

- **只提取实际看得到的特征。** 单点不能推断风格。
- **特殊用法是用户的声音。** 不规范的标点或表达——保留，不要修正。
- **样本冲突时**——优先最近 / 上下文最相似的文件。
- **样本稀少时**——能可靠提取的提取，剩下用默认。
- **风格校准只管语气与结构。** 不要把样本里的内容、claim、指标搬进 CV、报告或评估。
- **不能逐字复制或保留个人标识符。** 只存抽象的风格描述（语气 / 结构 / 词汇偏好）；不要 verbatim 引用样本句子；不要保留姓名、邮箱、电话等。

### 写回 _profile.md 的格式

抽取后写到 `modes/_profile.md`（**仅在至少有一个用户样本时**）：找到现有 `## Writing Style` 段并替换至下一个 `##` 标题（或文件结尾）的全部内容。如果不存在就追加。永远只保留一份。

```markdown
## Writing Style

_于 {date} 从 writing-samples/ 抽取。新增样本后重跑。_

**语气：** {例：口语化、自信、不含糊}
**句长：** {例：短促，平均 12 字}
**开头：** {例：动作优先 / 主语优先}
**标点：** {例：破折号作插入语，顿号一律保留，不用感叹号}
**词汇：** {例：偏好"做""跑""砍"而非"实施""领导""降低"}
**结构：** {例：散文为主，结论先行}
**声音：** {例："我做了"、主动语态为主，不用反问}
**避免：** {样本里完全没有的词或模式}
```

## 专业写作 & ATS 兼容

下面规则适用于**所有**进入候选人对外文档的文本：PDF Summary、bullet 点、求职信、表单回答、脉脉/微信消息。**不**适用于内部评估报告。

### 避免的中文 corporate-speak

- "勇于挑战""敢于突破""拥抱变化""迎难而上""赋能业务"
- "深度参与""充分论证""有效推动""扎实推进""强力执行"
- "全方位""多维度""一站式""端到端"（除非真的是端到端 ML pipeline）
- "复盘""抓手""闭环""沉淀""打透""穿透""对齐""拉齐"
- 直接抄 JD 的口号性表述（"以客户为中心""技术驱动业务"）

### Unicode 规范化

`generate-pdf.mjs` 会自动把中文引号、破折号、零宽字符规范化为 ATS 兼容形式。但生成时尽量直接用规范字符——避免：

- 全角字母数字（A→A、1→1）改成半角
- 中英文混排时，英文前后留空格："使用 Python 实现" 而非 "使用Python实现"
- 不要用 emoji（多数 ATS 解析失败）

### 句式多样化

- 不要每个 bullet 都用同一动词开头
- 长短句交替（短句。然后一句加上下文。再短一句。）
- 列表不一定都是"X、Y、Z 三件事"——有时两件，有时四件

### 具体优于抽象

- "P95 延迟从 2.1s 降到 380ms" 完胜 "提升性能"
- "Postgres + pgvector，12K 文档检索" 完胜 "设计可扩展的 RAG 架构"
- 在合规允许下，写出工具、项目、客户名

### CJK 字体回退链

`templates/cv-template.cn.html` 的字体策略：

1. `Noto Sans SC`（Google Fonts CDN——首选，效果最好）
2. `PingFang SC` / `Hiragino Sans GB`（macOS 系统字体）
3. `Microsoft YaHei`（Windows 系统字体）
4. `yoCareer CJK`（仓库内 `fonts/noto-sans-sc-subset.woff2` 兜底——常用 3500 字 + Latin 子集，~1.3MB）

GFW / 离线场景下走第 4 级，保证 PDF 不出框框字符。
